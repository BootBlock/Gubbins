# Phase 76 — Bulk edit & item clone (living plan)

Third feature-gap audit (`feature-gap-audit-2026-06-30c`) **Wave 1, candidate #3**. Productivity;
**~no migration** (`user_version` stays 1 — verified: all writes reuse existing columns/methods).

## Decisions (recommended defaults)

- **Bulk-editable fields:** category · location · condition · active-state · tags (add **or** replace).
  These reuse existing repository methods (`update` for category/condition, `move` for location,
  `softDelete`/`restore` for active, `TagRepository.setForItem` for tags) — no new write SQL.
- **Clone field policy:** copy descriptive/template fields (name + " (copy)", description, category,
  location, MPN, manufacturer, unit cost, condition, reorder thresholds, depreciation months,
  tracking mode + gauge shape, operational metadata) **+ custom-field values + supplier parts**;
  **strip** per-instance identity (serial no, batch/lot no, expiry, acquired-at, warranty-expiry,
  purchase price) and **reset quantity to 0** (gauge net to 0; serialised → one fresh instance).
- **Placement:** both actions live in the existing multi-select **selection bar** (no `ItemList` /
  `ItemDetailDialog` surgery). **Bulk edit** enabled when ≥1 selected; **Duplicate** enabled when
  **exactly one** selected (clone is inherently single-source). The selection already captures
  fields at toggle time (Phase 49/73 gotcha) and stores ids — reused as-is.

## Pure seams

### `src/features/inventory/bulk-edit.ts` (+ `bulk-edit.test.ts`)
- `BulkEditSpec` — wrapper-presence model so "change to None/cleared" is distinct from "leave
  unchanged": `{ category?: {value: string|null}; location?: {value: string};
  condition?: {value: Condition|null}; active?: {value: boolean};
  tags?: {mode: 'add'|'replace'; names: string[]} }`.
- `isBulkEditEmpty(spec)` — true when no field selected (disables Apply).
- `summariseBulkEdit(spec, lookups)` — human change lines for the confirm + aria-live.
- `parseTagInput(text)` — comma-split, trim, drop blanks (pure UI helper).
- `resolveItemTagNames(currentNames, tags)` — union (add) or replacement (replace), dedup
  case-insensitively. (`setForItem` re-dedups, but the seam is unit-tested independently.)

### `src/features/inventory/clone.ts` (+ `clone.test.ts`)
- `planItemClone(source: Item, options?)` → `CreateItemInput` (the strip/reset policy above; pure).
- `clonedSupplierPartInput(part: SupplierPart)` → `CreateSupplierPartInput` (preserves `isPreferred`).
- `clonedFieldValues(fields: ResolvedItemField[])` → `Record<fieldId, string>` (stored values only).

## Glue (mutations + UI)

- `mutations.ts`:
  - `useBulkEditItems` — `{ ids, spec }` → applies per item via existing repo methods; returns
    `{ succeeded, failed }`. Invalidation-based (not optimistic — multi-field, multi-item).
  - `useCloneItem` — `{ sourceId }` → reads source + field values + supplier parts, plans the seed,
    `create` (or `createSerialised` count 1 for SERIALISED), copies field values + supplier parts +
    operational metadata (post-create `update`). Returns the new `Item`. Invalidates items+locations.
- `BulkEditDialog.tsx` — Foundry `Modal` + `FormField`/`Select`/`Input`/`LocationSelect`; per-field
  **enable checkbox** + control; Apply/Cancel; an aria-live result region (Phase 63). Tokens only.
- `InventoryScreen` selection bar: **Bulk edit** + **Duplicate** buttons; an aria-live region
  announcing the bulk/clone outcome.

## Verification

`npx tsc -p tsconfig.app.json --noEmit`, `npm run test:run`, `npm run build` — all green.

## Out of scope / deferred (tracked)

- Bulk-edit of arbitrary custom fields / price / reorder thresholds — Backlog (start with the five).
- Open the clone in the detail dialog immediately after creating it — Backlog (it appears in the
  list; the selection bar announces success). Trigger: a request to edit the copy inline.
- **Clone is best-effort, not atomic** — the create + child copies (metadata / custom fields /
  supplier parts) are separate transactions spanning repositories, so a mid-sequence failure can
  leave a partially-populated clone (documented on `useCloneItem`; the stray copy is easily
  soft-deleted). A cross-repository transactional clone is Backlog. Trigger: a need for guaranteed
  all-or-nothing duplication.
