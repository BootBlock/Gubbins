# Phase 75 — Unified "Upcoming" calendar / agenda (living plan)

Third feature-gap audit (`feature-gap-audit-2026-06-30c`) **Wave 1, candidate #1**. Read-only;
**no migration** (`user_version` stays 1). Aggregates every date-driven event into one
chronological, time-ordered agenda — today these are scattered across the alert centre and
dashboard widgets, with no single time-ordered view.

## Decisions (recommended defaults, none load-bearing enough to block)

- **Agenda/list with date buckets**, not a calendar grid (§2.4.3 — simpler, mobile-first, no
  date-grid dependency). Buckets, in order: **Overdue · Today · This week · This month · Later**.
  The "Later" bucket is a catch-all so nothing is ever hidden.
- **New screen + nav entry** at `/upcoming` ("Upcoming"), following `AlertsScreen`/`ReportsScreen`
  (header + `<main id={MAIN_CONTENT_ID}>` + a Phase-63 always-mounted `<LiveRegion>` completion
  region). A compact dashboard widget is **out of scope** for this phase (the nav entry suffices;
  the dashboard already has per-source widgets) — noted, not built.
- **Kind filter**: a token-styled segmented control to toggle kinds on/off (all on by default).
- Every event carries a **jump-to-source** link (mostly `/inventory?item=…` like the alert centre's
  `target`, plus checkouts → `/inventory`). Reuse the alert centre's `AlertTarget` shape.

## Event kinds & sources (all existing queries / read-only)

| Kind | Source (existing) | dueAt | Notes |
| --- | --- | --- | --- |
| `maintenance` | `MaintenanceRepository.listUpcoming(now)` (**new read-only method**, = `listForItem` for all active items) | TIME: `maintenanceDueAtMs(...)`; USAGE: `now` when due, else excluded | USAGE has no calendar date → only surfaced when currently due (`hasDate=false`). |
| `warranty` | `ItemRepository.listWarrantyExpiring(BIG, now)` | `Date.parse(warranty_expires_at)` | Big window (≈100 yr) → all future + overdue; method already includes past. |
| `expiry` | `ItemRepository.listExpiringWithin(BIG, now)` | `items.expiry_date` | Big window → all future + already-expired (method is `<=` cutoff). |
| `checkout-due` | `CheckoutRepository.listOpen()` | `checkouts.due_date` | Only open loans with a `due_date`. |
| `reorder` | `ReportRepository.listReorderShortfall()` | `now` (`hasDate=false`) | "Reorder now" — a present state, not a date; lands in Today. |

`listUpcoming` is the only new repository method (read-only SELECT, no schema change). `listOpen`,
`listReorderShortfall`, `listExpiringWithin`, `listWarrantyExpiring` already exist.

## Pure seam — `src/features/calendar/agenda.ts` (+ `agenda.test.ts`)

Logic out of the glue (house pattern), `now` injected, exhaustively unit-tested:

- Types: `AgendaKind` (`'maintenance' | 'warranty' | 'expiry' | 'checkout-due' | 'reorder'`),
  `AgendaBucket` (`'overdue' | 'today' | 'week' | 'month' | 'later'`), `AgendaEvent`
  (`id, kind, title, detail, dueAt: number, hasDate: boolean, target`), `AgendaSources`.
- `buildAgenda(sources, now): AgendaEvent[]` — one lane builder per kind; flatten + stable sort
  by `dueAt` asc then `id`. Date-less actionable events (USAGE maintenance due, reorder-now) get
  `dueAt = now`, `hasDate = false`.
- `bucketAgenda(events, now): { bucket, label, events }[]` in fixed order. Boundaries from a pure
  `startOfLocalDay(now)` helper (TZ-robust: tests derive event times from the same anchor):
  - **overdue**: `dueAt < now`
  - **today**: `now ≤ dueAt < startOfLocalDay(now) + 1 day`
  - **week**: `… < startOfLocalDay(now) + 7 days`
  - **month**: `… < startOfLocalDay(now) + 30 days`
  - **later**: otherwise
- `filterByKind(events, enabledKinds)` for the UI filter.
- Reuses `maintenanceDueAtMs` (already exported from `@/features/alerts/alerts`) for TIME due.
