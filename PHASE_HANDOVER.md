## 2. Current database schema snapshot (`PRAGMA user_version = 4`)

Per-connection pragma on every open: `PRAGMA foreign_keys = ON;`. All tables `STRICT`.
`SQL_NOW_MS` (UNIX-ms) is exported from `@/db/migrations`. v1 = `app_meta`; v2 = core domain;
v3 = category schemas/tags/images/attachments + `items.serial_no`; **v4
(`src/db/migrations/v4-projects.ts`)** adds projects, BOM lines, item aliases, the
`items.mpn`/`manufacturer`/`unit_cost` columns, and the system-locked **"In Transit"** location.
Every syncable table keeps the §7.1 UUID PK + `updated_at` auto-stamp trigger (the
`WHEN NEW.updated_at = OLD.updated_at` LWW pass-through guard).

**Phase 4 additions only** (v1–v3 tables unchanged — see git history / prior handover):

```sql
-- v4: items gains BOM-match keys + costing (all additive ALTER ... ADD COLUMN)
ALTER TABLE items ADD COLUMN mpn TEXT;            -- Manufacturer Part Number (auto-match key)
ALTER TABLE items ADD COLUMN manufacturer TEXT;
ALTER TABLE items ADD COLUMN unit_cost REAL;      -- current replacement value per unit
CREATE INDEX idx_items_mpn ON items(mpn COLLATE NOCASE);

-- v4: supplier/alternative part identifiers mapped to a local item (§4 Universal Alias Mapping)
CREATE TABLE item_aliases (
  id TEXT PRIMARY KEY NOT NULL,
  item_id TEXT NOT NULL REFERENCES items(id) ON DELETE CASCADE,
  alias TEXT NOT NULL,
  updated_at INTEGER NOT NULL DEFAULT (<SQL_NOW_MS>)
) STRICT;
CREATE UNIQUE INDEX idx_item_aliases_alias ON item_aliases(alias COLLATE NOCASE); -- unambiguous match
CREATE INDEX idx_item_aliases_item_id ON item_aliases(item_id);

-- v4: projects (§4 Projects & BOMs)
CREATE TABLE projects (
  id TEXT PRIMARY KEY NOT NULL, name TEXT NOT NULL, description TEXT,
  status TEXT NOT NULL DEFAULT 'PLANNING',              -- PLANNING|ACTIVE|COMPLETED|ARCHIVED
  costing_mode TEXT NOT NULL DEFAULT 'CURRENT_REPLACEMENT', -- CURRENT_REPLACEMENT|POINT_IN_TIME
  created_at INTEGER NOT NULL DEFAULT (<SQL_NOW_MS>),
  updated_at INTEGER NOT NULL DEFAULT (<SQL_NOW_MS>),
  CHECK (status IN ('PLANNING','ACTIVE','COMPLETED','ARCHIVED')),
  CHECK (costing_mode IN ('CURRENT_REPLACEMENT','POINT_IN_TIME'))
) STRICT;

-- v4: BOM lines — reservation + procurement state live on the line (§4)
CREATE TABLE project_bom_lines (
  id TEXT PRIMARY KEY NOT NULL,
  project_id TEXT NOT NULL REFERENCES projects(id) ON DELETE CASCADE,
  item_id TEXT REFERENCES items(id) ON DELETE SET NULL,  -- NULL = unmatched/manual line
  designator TEXT, mpn TEXT, manufacturer TEXT, description TEXT,
  required_qty INTEGER NOT NULL DEFAULT 1,
  reserved_qty INTEGER NOT NULL DEFAULT 0,
  reservation_status TEXT NOT NULL DEFAULT 'NONE',  -- NONE|TENTATIVE|ACTUAL
  procurement_status TEXT NOT NULL DEFAULT 'NONE',  -- NONE|ORDERED|IN_TRANSIT|RECEIVED
  unit_cost_snapshot REAL,                          -- point-in-time cost captured at add time
  position INTEGER NOT NULL DEFAULT 0,
  created_at INTEGER NOT NULL DEFAULT (<SQL_NOW_MS>),
  updated_at INTEGER NOT NULL DEFAULT (<SQL_NOW_MS>),
  CHECK (required_qty >= 0), CHECK (reserved_qty >= 0),
  CHECK (reservation_status IN ('NONE','TENTATIVE','ACTUAL')),
  CHECK (procurement_status IN ('NONE','ORDERED','IN_TRANSIT','RECEIVED'))
) STRICT;
CREATE INDEX idx_project_bom_lines_project_id ON project_bom_lines(project_id, position);
CREATE INDEX idx_project_bom_lines_item_id ON project_bom_lines(item_id);
```

**Triggers:** `trg_<table>_updated_at` AFTER UPDATE auto-stamp added for `item_aliases`,
`projects`, `project_bom_lines`. **No new protect triggers were needed for "In Transit"** — it is
seeded with `is_system = 1`, so the existing v2 `trg_locations_protect_system_update/delete` guards
already make it immutable/undeletable.
**System-location sentinels:** `UNASSIGNED_LOCATION_ID = 00000000-0000-4000-8000-000000000001`;
**`IN_TRANSIT_LOCATION_ID = 00000000-0000-4000-8000-000000000002`** (`IN_TRANSIT_LOCATION_NAME = 'In Transit'`).
**New `item_history.action` values (no CHECK on that column — append-only enum in `constants.ts`):**
`RESERVED`, `RESERVATION_CLEARED`, `PROCURED`, `RECEIVED`, `CONSUMED`, `ASSEMBLED`.

**Adding Phase 5 schema:** create `src/db/migrations/v5-*.ts` (version `5`), register in
`src/db/migrations/index.ts`. Never edit shipped migrations; use the §2.3.3 12-step pattern for
non-additive changes. New syncable tables replicate the UUID PK + `updated_at` trigger pattern.
(FTS5 virtual tables for Phase 5 search are the obvious next addition — see §7.)

## 3. Repository layer (`src/db/repositories/`)

All SQL lives here over the injected `IDatabaseDriver` (§2.1.1) — **components never write SQL**.
Construct `new XRepository(driver, { isWriteSuspended? })`. **Tests inject `createMemoryDriver()`**
(`node:sqlite`, §8.5.2); production singletons (`getItemRepository`, `getLocationRepository`,
`getCategoryRepository`, `getTagRepository`, `getImageRepository`, `getAttachmentRepository`,
**`getProjectRepository`**) wire the worker driver + the storage Hard-Stop gate. Reads paginate
(`Page<T>`, LIMIT/OFFSET ≤ 100). Growth-writes call `assertWritable()`; deletes bypass it.

### ProjectRepository (NEW — `ProjectRepository.ts`)
```ts

## 4. BOM import parser (`src/features/projects/bom-import.ts`) — §4, pure & native

- **`parseCsv(text) : string[][]`** — dependency-free RFC-4180-ish reader: quoted fields, doubled-quote escapes, embedded commas/newlines, CRLF/LF, trailing-blank-line tolerance.
- **`parseBom(text) : ParsedBomLine[]`** — maps common **KiCad/generic** headers (Reference/Designator, MPN/Mfr Part Number/Part Number, Manufacturer/Mfr, Description/Value/Comment, Quantity/Qty) case- & whitespace-insensitively; qty defaults to 1; description falls back to Value; throws `BomImportError` on empty/unrecognised input.
- Auto-matching is **not** in the parser — the `useImportBom` hook calls `ItemRepository.findByMatchKey(line.mpn)` per line, then `addLine` (matched → item-linked, else manual). Fully unit-tested on `:memory:` (parser is pure; no DB/React).

## 7. Technical debt, stubs & deferrals

1. **FTS5 / Visual Builder (Phase 5, next):** `ItemRepository.list({search})` is still a `LIKE` scan; FTS5 + the Visual-Builder AST (§5.1) are Phase 5. The `:memory:` driver doesn't exercise FTS5 — keep the real-browser smoke as the FTS5 guard. Capabilities (weighted tagging) are also Phase 5.
2. **Reservations model:** reservations are **ledger annotations on the BOM line** — they do **not** decrement an item's on-hand `quantity` (which represents physical stock). Shopping-list shortfall = `required − reserved`. There is no cross-project "available vs committed" stock view yet; if Phase 9 (logistics) needs true allocation, revisit.
3. **Procurement / In-Transit:** modelled as a per-line `procurement_status`. `receiveLine` brings stock in for **matched DISCRETE** items only (gauge/serialised/unmatched lines just flip to RECEIVED). The system-locked **In-Transit location exists** but incoming stock is not auto-manifested as placeholder items there (spec offers status *or* location; we use status). `ORDERED`→`IN_TRANSIT`→`RECEIVED` transitions are free (not a strict state machine).
4. **BOM costing:** `unit_cost_snapshot` is captured at `addLine` time and never auto-refreshed; `CURRENT_REPLACEMENT` always reads live `items.unit_cost`. Currency is a single base currency (no FX).
5. **Aliases:** `item_aliases` is a Phase-4 down-payment on §8 Universal Alias Mapping (used for BOM auto-match). No alias-management UI yet (set via repository/import); add one if Phase 8 needs it.
6. **Export Wizard:** deferred to Phase 6 per §5 (confirmed with the developer). The shopping list renders in-app only — no CSV/Markdown export yet.
7. **Sync (Phase 7):** projects/lines/aliases carry UUID PKs + `updated_at` triggers, ready for LWW. `item_tags` and `item_aliases`/BOM joins still need Phase 7 to decide M:N sync semantics; tombstones/NTP/delta-CRDT/re-parenting remain Phase 7.
8. **Theme toggle** still dark-only (persisted but not applied to `<html>`). **Bundle** main chunk > 500 kB (warning only; `projects` is a separate 26 kB route chunk).
9. **Tooltip keyboard-link reach** (focus leaving the trigger closes the bubble) — unchanged Phase 3 debt.
